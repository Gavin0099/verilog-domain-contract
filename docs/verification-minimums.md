# Verification Minimums

## Minimum Disclosure Required In Output

- what was checked
- what was not checked
- why unchecked areas matter
- next verification actions

## Non-Equivalent Evidence

The following do not equal design completion:

- compile success only
- lint clean only
- single happy-path testbench pass

## Claim Boundaries

- no "verified implementation" claim without sufficient stated evidence scope
- draft/candidate claims must include residual risk disclosure
