1. General Description 
The RTS5264 is a PCI Express revision 4.0 compliant card reader controller that integrates a PCI Express 
Transceiver, regulators, memory card access units, and a high speed multiplexer into a single chip. The 
RTS5264 can support Secure Digital TM, Secure Digital High Capacity TM (SDHC), Secure Digital eXtended 
Capacity TM (SDXC), and Secure Digital Ultra Capacity TM (SDUC).   
The chip fully supports SD UHS-I (Ultra High Speed-I), UHS-II (Ultra High Speed-II), DDR200, DDR225 
and SD Express mode. The RTS5264 only requires one PCI Express lane. 
The chip integrates 3.3V-to-1.8V and 3.3V-to-1.2V regulators and MOSFET for card power switch, which 
could dramatically reduce the system BOM cost. 
The chip is fully compliant with modern platform requirements for the card reader applications. It supports 
LTR (Latency Tolerance Reporting), and L1 substate and RTD3 (Runtime D3) functions. It supports 
Windows modern standby. 
The chip is fully compliant with 3.3V/1.8V/1.2V PCIe sideband si gnal (PERST#, CLKREQ#, WAKE# , 
PRSNT#). 
The chip VID/PID is 10EC/5264.

5. Product Features 
SD Express/SD7.x/SD8.0 
SD Express merges PCIe and NVMe interfaces into SD memory cards, together with legacy SD interface. 
An SD card with PCIe interface is known as an SD Express card.  
SD Express was proposed in SD7.x and given enhanced speed in SD8.0.  
 SD7.x: SD Express can achieve speeds of up to 985MB/s by PCIe Gen3 1-lane 
 SD8.0: SD Express has more speed combinations, and can support PCIe Gen3 2-lane (1969MB/s), 
PCIe Gen4 1-lane (1969MB/s), or PCIe Gen4 2-lane (3938MB/s) 
There is one design for SD7.x , and three different designs for SD8.0. Select the correct option to fit your 
design.  
 
 
SD Ultra Capacity (SDUC) 
The old SDXC spec was limited to 2TB . The SDUC can support more address es and extends the size to 
128TB for legacy SD interface.

PCI Express 
The PCIe Gen3 8Gbps l-lane can provide up to 985MB/s bandwidth (Theoretically).  The PCIe Gen3 8Gbps 
2-lane can provide up to 1969MB/s bandwidth ( Theoretically). PCIe Gen4 l-lane can provide up to 
1969MB/s bandwidth ( Theoretically). PCIe Gen4 2 -lane can provide up  to 3938MB/s bandwidth 
(Theoretically). 
LTR (Latency Tolerance Report) 
LTR is an active message from device to root, which will report device behaviors. Regarding its tolerance 
of read/write service latencies. That can help system to estimate whole system service routine and enhance 
efficiency of performance/power saving. 
L1 Sub-State 
L1 sub-state is a new approach for power management of PCIe interface s, and is used to enhance power 
saving effects. It can enter a deeper power saving state than traditional L1, called L1.0.  
When the L1.0function enabled, the root and device will use the sideband signal  CLKREQ# to negotiate 
the L1 sub-state entry and exit and allow the device the opportunity to turn off most of the PCIe block. 
RTD3 
Traditionally the system po wer state indicated the whole system’s components power state.  RTD3 is 
provided to allow the device power state to vary from the system power state. E.g., the device can be in a 
D3 state when system is in an S0 state.  
The system could provide individual p ower to control each devices state. On a wakeup from RTD3 state, 
the device needs to connect its Wake# to the system. If the system implements individual power for each 
device, then AUX-power is required for support  D3cold state. However, Wake# and Aux-Power both are 
required to support RTD3. 
Application Performance Class 
The Application Performance Class enables higher random performance in an SD card. The SD Association 
has defined Application Performance Class 1 (A1) and Application Performance Class 2 (A2) standards. 
An A1 card should meet at least 1500  Input-Output access Per Second (IOPS) in read  and 500 IOPS in 
write.  
To further enhance the random performance, Class 2 (A2) introduces new technology such as command 
queue (CQ) and cache. An A2 SD card should meet 4000 IOPS in read and 2000 IOPS in write.  
DDR200/DDR225 
DDR200/DDR225 is an enhanced mode of SD card. DDR200/DDR225 is not  approved by the SD 
Association, but some card vendors uses this mode as an ‘enhanced mode’. It can provide more bandwidth 
than UHS-I SDR104, but requires more care in the PCB layout.

9. Notes for Implementation 
9.1. PCIe Hot Plug 
In order to use this chip, system support for PCIe hot plug is necessary. The SoC or PCH has to support 
PCIe hot plug. BIOS also has to enable PCIe hot plug related option. As different platforms have different 
settings, please contact your SoC or PCH/BIOS vendor for detail.  
 
9.2. PRSNT# 
In this chip, the PRSNT# pin is used for hot plug notification. You must connect this pin to the SoC, PCH, 
or EC  GPIO. The GPIO connections depend on your system design.  Please contact your BIOS or EC 
engineer for details.  
We strongly suggest to configure your SoC, PCH, or EC GPIO to input mode. We suggest to use the same 
power source as AUX3V3 to pull up PRSNT#. 
When the chip drives PRSNT# low , it indicates that a new device has been plugged in. When PRSNT# 
goes high, it means a device was unplugged. 
We recommend to implement PRSNT# for hot plug detection. 
Table 79. PRSNT# 
PRSNT# Description 
1 Device was not presented 
0 Device was presented 
 
9.3. PCIe 1.8V/1.2V Sideband IO 
In SD Express mode,  the SD Express card acts as a standalone PCIe device. As SD Express cards come 
from various vendors, we cannot ensure all SD Express cards support 1.8V/1.2V PCIe sideband IO. For the 
best compatibility, please follow the instructions below. We provide 3 options for customer reference. 
1. Add level shift for all PCIe sideband (PERST#, CLKREQ#, WAKE#). 
2. Add level shift for CLKREQ# only. 
3. If BOM cost is very important for your project, you can disable the CLKREQ# feature for SD Express 
cards. Refer to BIOS guide 0x814 bit 1 for detail. However, if the SD Express card supports L1 substate, it 
may result in the SD Express card failing to enter L1 substate, so the system power consumption might be 
higher when using the SD Express card. 
In legacy SD mode, this chip can fully support PCIe 3.3V/1.8V/1.2V sideband IO.

SD7.x and SD8.0 follow different hardware PCB designs and SoC/PCH/CPU settings, as well as different 
SD sockets.  
SD7.x design is a PCIe Gen3 1-lane design. There is only one option in the SD7.x design. The SD8.0 design

Please select the correct option to fit your design.  
 
9.5. UHS-II Polarity Cannot be Reversed 
The polarity of SD_HSIP and SD_HSIN  cannot be reversed . The polarity of SD_HSOP and SD_HSON  
also cannot be reversed.  
 
9.6. Micro SD design 
Note that micro SD Express card and micro  SD UHS-II card pin location s are differ ent. Some socket  
vendors provide a ‘combo’ socket. They claim that their socket can support both micro SD Express and 
micro SD UHS-II card in one micro SD socket. The socket vendor should ensure their combo socket quality. 
Also note that micro SD does not have SD_WP#. It should connect to GND for a normal type socket. Refer 
to the application for more details.
